     mutation method, so write-write conflict detection (`WriteConflict`)
     can't actually trigger through any real code path today — a
     separate, larger feature.
   - `commit_current_tx()`'s `WriteConflict` result still isn't surfaced
     to wire-protocol clients (a commit that lost a conflict looks
     identical to a client as a real commit).
   - (Forward-looking, not a bug today) If Phase 8h ever implements real
     `PROMOTE REPLICA` (currently a stub in `graft-server/src/admin.rs`),
     chained replication could reintroduce a tx_id collision surface
     between two clusters that both start their `REPLICA_TX_ID_BASE`
     counter at the same value — flagged for whoever builds that phase.

## Key Decisions Already Made (don't re-litigate)

From Milestone 1 — still binding:
- Per-replica outbox fan-out lives INSIDE `ReplicationSender::poll()`.
- Replica identity/cluster_id resolved once per process, not per-connect.
- No read timeout on `primary_reader` (asymmetric with replica's timeout,
  intentionally).
- No full WAL-replay catch-up for a reconnecting replica — that's Phase
  8g, not these 15 findings.

From Milestone 2 — still binding:
- LSN persistence uses a reservation-ceiling allocator
  (`REPL_LSN_RESERVE_BATCH = 10_000`), NOT "persist on WAL group-commit
  cadence".
- Corrupted-batch handling poisons the `ReplicationReceiver` in-memory and
  halts forward progress; does NOT force a TCP reconnect. Poisoning does
  NOT survive a replica process restart (known, accepted gap — Phase 8g's
  to solve).

From Milestone 3 — still binding:
- Fixed #7 via disjoint reserved tx_id ranges (`REPLICA_TX_ID_BASE = 1 << 63`
  in `crates/graft-runtime/src/cluster.rs`), NOT via a larger gap or a
  snapshot-LSN redesign (the two options raised in the prior handoff) — the
  disjoint-range approach makes collision structurally impossible rather
  than merely improbable, and eliminates the sync mechanism entirely rather
  than narrowing its race window. Full rationale in `tasks/todo.md`.
- No `is_replica` flag was added to `ShardCluster` — nothing needs to
  branch on role anymore once construction seeds the right tx_id base.
- #9's fix ships an explicit Abort record on WriteConflict, NOT silence —
  shipping nothing would leak the tx_id in `CommitBuffer.pending` forever
  (`CommitBuffer::append` only resolves a buffered tx on seeing an explicit
  terminal Commit or Abort record).

## Pitfalls to Avoid

- `graft-server` has no `[lib]` target — integration tests under `tests/`
  can't reach its private modules. Use `#[cfg(test)] mod tests` inside the
  relevant `src/*.rs` file instead.
- Before assuming a `cargo clippy -- -D warnings` failure is your fault,
  `git stash && cargo clippy ... && git stash pop` to check if it pre-dates
  your change (has been true before — toolchain drift).
- `crates/graft-io/src/uring.rs`'s `is_data_file()` opens anything not
  named `*.wal` or `*.meta` with O_DIRECT on Linux — check this function
  before adding any new small/unaligned persisted file. This dev machine is
  macOS, so `uring.rs` (`#[cfg(target_os = "linux")]`-gated) can only be
  verified by reading, not compiling/running — flag any changes there
  explicitly.
- RefCell field borrows in `Shard`'s methods don't compose across method
  calls even for disjoint fields — a function-scoped (not block-scoped)
  `self.wal.as_ref().map(|w| w.borrow_mut())` needs an explicit `drop()`
  before calling another `&mut self` method. Full explanation in
  `tasks/lessons.md`.
- A TOCTOU race can sometimes be eliminated by construction (disjoint
  reserved value ranges) instead of narrowed by more careful
  synchronization — check whether the two racing things actually need to
  share a value space before reaching for "add more syncing." Full
  writeup in `tasks/lessons.md`.
- A regression test can validate a fix's logic even when the bug's
  real-world trigger path doesn't exist yet in the codebase (e.g. calling
  a private lower-level method directly from a same-module test to
  construct the precondition) — say so explicitly rather than skipping
  coverage or expanding scope to build the missing trigger path. Full
  writeup in `tasks/lessons.md`.
